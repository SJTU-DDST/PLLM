# PLLM HiberAgent 演示视频脚本

目标时长 2 分 40 秒。所有角标使用 `LIVE`、`MOCK-VALIDATED` 或 `HISTORICAL REPLAY`。

## 0:00-0:18 问题

画面：Nemotron 流式生成 CUDA kernel，右侧 `nvidia-smi` 显示模型常驻。启动 Blender render 前停顿一秒。

旁白：DGX Spark 可以把 100B 模型放在桌面，但桌面 GPU 还属于正在创作和游戏的用户。传统 vLLM 不知道前台任务什么时候需要资源。

## 0:18-0:42 前台感知

画面：打开 Vue 控制中心。焦点切到 Blender，Foreground task 与 GPU/NVENC 指标变化；Expert Residency 面板从 512 收缩到 256/128 slots，并始终显示 `CONTROL PLANE ONLY`。状态从 `ACTIVE` 进入 `YIELDING`。

旁白：PLLM 不只看总利用率。它同时读取 GNOME 焦点、进程级 GPU、UMA 内存压力和供电状态，在物理数据面可用时选择精确专家驻留，否则先在 token 边界冻结 scheduler。当前回放中的专家数字是规划值，不冒充实际释放。

## 0:42-1:08 快速释放

画面：Blender 压力上升，状态进入 `QUIESCING -> HIBERNATED`；四级内存流从 GPU 指向 HiberCache/NVMe。并排展示释放 GiB 与 `MemAvailable`。

旁白：明确内存压力触发 Level 2。不可变的 75GB 权重直接丢弃，不重复写 SSD；可复用 Attention block 进入 HiberCache，混合状态允许由 token 重算。

## 1:08-1:28 前台 QoS

画面：Blender benchmark 曲线对比常驻 vLLM 与 PLLM。只显示真实实验 JSON。

旁白：核心指标不是“模型睡着了”，而是前台吞吐恢复到无后台模型基线的多少。目标是 90% 以上，并同时报告功耗。

## 1:28-1:55 同流恢复

画面：关闭 Blender；30 秒迟滞可在视频中倍速。状态进入 `RESTORING`，显示 fastsafetensors shard 进度；代码回答从“paused at token N”继续。

旁白：SparkLoad 从本地 NVMe 或 ConnectX-7 host staging 恢复。请求不重新提问，恢复后继续同一输出流。屏幕同时展示 exact-match 校验。

## 1:55-2:18 DGX Spark 适配

画面：capability probe 显示 GB10 UMA、GPUDirect RDMA false、host staging ready；切到 C++ MR benchmark JSON。

旁白：DGX Spark 不支持 GPUDirect RDMA。PLLM 按 NVIDIA 指南使用 cudaHostAlloc 注册 MR，不把当前服务器的 GDR 能力移植成虚假结论。网络和 host-copy 指标始终分开。

## 2:18-2:40 工程与结尾

画面：测试证据面板明确分成“数据面修改前 39 passed”和“新增数据面 IMPLEMENTED / UNTESTED”，再展示桌面悬浮窗、策略编译器、工程迭代与审稿答辩目录。不要在本轮录制中展示尚未执行的 CMake、RDMA、Marlin 或 GPU 测试绿灯。

旁白：这是一个 vLLM 可执行控制面、一套已落到源码但仍待 GPU 验收的数据面，以及一个诚实展示降级路径的桌面产品。PLLM 让本地 AI 在用户需要 GPU 时主动让路，再带着原任务回来。
